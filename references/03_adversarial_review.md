# 对抗审查

## 核心机制
不要只从一个视角看自己的设计。依次切换以下四个角色，每个角色独立给出批评。

## 审查角色

### 1. 目标受众
- 这页我能一眼看懂核心观点吗？
- 有没有让我困惑的术语或数据？
- 这页在帮我达到演示目标吗？

### 2. 挑剔设计师
- 视觉是否统一？配色、字体、留白是否协调？
- 有没有过度装饰？
- 信息层级是否清晰？

### 3. 逻辑审查者
- 这一页和上一页是否连贯？
- 论据是否支撑观点？
- 有没有冗余或遗漏？

### 4. 时间管理者
- 这页在总时长里占多少？
- 有没有某页信息过载？
- 结尾是否有明确的行动号召？

## 输出格式
```markdown
## 审查报告

### High（必须修复）
1. [角色] [问题] [修改建议]

### Medium（建议修复）
1. ...

### Low（可选）
1. ...
```

## 评分规则
- 任一 High 问题未修复 → 必须迭代
- Medium 问题超过 3 个 → 建议迭代
- 无 High 且 Medium ≤ 3 → 可以通过
