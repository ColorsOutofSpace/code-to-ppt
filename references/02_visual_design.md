# 视觉设计系统

## 设计原则
- **内容驱动设计**：配色、字体、风格不是随机选的，必须反映演示主题和受众。如果把这些配色换到另一个完全不相关的主题上依然"能用"，说明选择不够具体。
- **视觉为叙事服务**：每一页的视觉元素都在帮助传达一个核心观点，而不是装饰
- **一页一个核心观点**：不要试图在一页里塞多个论点
- **信息密度适中**：字少、图大、留白足
- **去 AI 味**：刻意避开"烂大街的 AI 视觉特征"。见下方专项说明

## 去 AI 味设计（Anti-AI-Flavor）

**核心：好的 PPT 看起来像设计师做的，不像 ChatGPT 生成的。**

LLM 有非常强的"风格惯性"，如果不刻意避开，生成的 PPT 会充满以下特征：

### 禁止的视觉特征

| AI 味特征 | 为什么烂 | 替代方案 |
|---|---|---|
| **紫蓝青渐变**（#667eea → #764ba2 之类） | LLM 默认色板，已被滥用 100 万次 | 用 dominance 60-30-10 中真正反映主题的配色 |
| **emoji 满天飞**（✨🚀💡🎯 装饰用） | 廉价感、儿童向 | 真正的图标（Material Design / SVG） |
| **居中粗体大标题 + 居中正文** | 模板感、缺乏设计意图 | 正文左对齐，标题可居中或左对齐 |
| **三段式平行结构**（"问题 / 方案 / 价值"） | 套话，论证无力 | 用金字塔结构，每页只支撑一个论点 |
| **SmartArt 风格箭头流程图** | Office 2010 遗物 | 真正的数据流图 / 时间线 / 矩阵 |
| **AI 套话**（赋能、抓手、闭环、打造、致力于、深入贯彻） | 空话，无具体内容 | 替换为"让 X 在 Y 场景下做到 Z" |
| **玻璃拟物 / 模糊渐变卡片** | 模板感、缺乏骨架 | 用 dominance 的硬色块 + 明确边框 |
| **金句大空白页**（每 3 页来一次喘息页） | 节奏感被破坏 | 真正的转场页或视觉母题强化页 |
| **占位图框 + 灰色 icon** | 偷懒、不专业 | 真正的图（图表 / 照片 / 数据可视化）|
| **每页都是"1 大标题 + 3 要点 + 1 icon"** | 单一节奏，视觉疲劳 | 5 种页面类型混用，每页有变化 |

### 反 AI 味检查清单

生成前自检：
- [ ] 配色**不是**紫蓝渐变、不是玻璃拟物、不是默认主题？
- [ ] 字体**不是**默认 Arial / Calibri / 思源黑体 Regular？
- [ ] 页面上**没有**装饰性 emoji？
- [ ] 标题**没有**全部居中？
- [ ] 要点**没有**清一色"3 个 bullet + 1 个 icon"？

生成后审查（挑剔设计师必查）：
- [ ] 整体看起来像设计师做的，不像 ChatGPT 套模板？
- [ ] 把 deck 投屏给陌生人，对方能不能看出"这是 AI 生成的"？如果能，需要重做

> **去 AI 味的本质**：让设计**有人味**——有具体的立场、具体的视觉选择、具体的母题。不是"看起来有设计感"，而是"看起来这个人/这个团队有自己的设计语言"。

## 内容驱动设计（Content-Informed Design）

做视觉方案前，先回答三个问题：
1. **这个主题是什么调性？**（严肃学术 / 科技前沿 / 温暖人文 / 商业硬核）
2. **如果听众闭上眼睛，应该感受到什么情绪？**（信任 / 兴奋 /  calm / 紧迫）
3. **这套配色换到另一个主题上是否依然成立？** 如果是，说明还不够贴合。

> 不要给所有主题用同一套默认配色。为每个具体场景重新设计。

## Dominance 原则（60-30-10 法则）

- **60% 主导色**：背景、大面积色块，奠定整体氛围
- **30% 辅助色**：标题栏、分区背景、次要信息
- **10% 强调色**：核心数据、CTA、关键结论——只做点缀，绝不泛滥

**不要给所有颜色同等权重。** 必须有一个绝对主导的视觉角色。

## 视觉母题（Visual Motif）

选一个**单一 distinctive 元素**，贯穿全篇重复使用：
- 圆角矩形边框 + 统一圆角半径
- 左侧粗色条（3-5pt）标记重点页
- 图标统一放在彩色圆形底中
- 图片统一用某种形状遮罩（圆形、圆角矩形）

> 母题的作用是"让不相关的几页看起来属于同一个 deck"。不要每页换花样。

## 三明治结构（明暗节奏）

两种节奏可选：
- **经典三明治**：封面深色 → 内容浅色 → 结尾深色。深色负责"锚定注意力"，浅色负责"承载信息"。
- **全深色**：从头到尾保持深色背景，适合科技感、高端感。需要更强的对比度和字体粗细对比。

不要随机混用深浅——节奏混乱等于没有节奏。

## 页面类型

| 类型 | 用途 | 结构 |
|---|---|---|
| Hero | 开场/章节转场 | 大标题 + 一句话 + 大背景色块 |
| Point | 阐述一个观点 | 标题 + 3 个以内要点 + 可选图标 |
| Chart | 展示数据 | 标题 + 图表 + 一句洞察 |
| Quote | 强调金句 | 引用文字 + 出处 |
| Closing | 结尾 | 核心信息 + 行动号召 |

## 配色方案（示例，必须根据具体主题重新设计）

> 下面的三套只是起点。真正好的配色必须回答"这个主题是什么情绪"。

- **专业沉稳**（学术、B2B、金融）：深蓝 `#1a365d`（60%）+ 浅灰 `#f7fafc`（30%）+ 砖红 `#c53030`（10%）
- **科技现代**（AI、SaaS、开发者）：深黑 `#0d1117`（60%）+ 青绿 `#00d9ff`（30%）+ 白色 `#ffffff`（10%）
- **温暖亲和**（教育、公益、消费）：米白 `#faf9f6`（60%）+ 砖红 `#c05621`（30%）+ 深蓝 `#2c5282`（10%）

**选择标准**：
- 深色背景 → 需要更强的对比度（文字至少 WCAG AA，最好是 AAA）
- 浅色背景 → 可以放松对比度，但要注意区分层级
- 强调色只用在一个地方：核心数据、CTA、结论句

## 字体搭配

不要默认 Arial。标题要有性格，正文要干净。

**中文场景**：
- 标题：思源黑体 Heavy / 阿里巴巴普惠体 Bold / 站酷高端黑
- 正文：思源黑体 Regular / 苹方 / 微软雅黑

**英文场景**：
- 标题：Montserrat Bold / Inter Bold / Roobert（有现代感）
- 正文：Inter Regular / Segoe UI / Open Sans

**搭配原则**：
- 标题和正文不要混用超过 2 个字体家族
- 中文和英文各自独立成对，不要中英文用同一个字体

## 一致性检查
- [ ] 全文字体不超过 2 个家族（中文1个+英文1个）
- [ ] 每页主色不超过 3 种，且严格遵循 60-30-10 权重
- [ ] 视觉母题在全 deck 中保持一致
- [ ] 标题层级统一（封面>章节>正文>注释）
- [ ] 页边距一致（建议上下 0.5"，左右 0.75"）
- [ ] 正文左对齐，标题可居中或左对齐，但不要混用
- [ ] 文字对比度满足 WCAG AA（深色背景尤其要严格）
