# 认知访谈（拷打引擎）

## 目标
把用户模糊的“帮我做个 PPT”变成清晰、可执行的设计约束。

## 必问问题清单

### 战略层
1. 这场演示的最终目标是什么？（说服 / 汇报 / 教学 / 融资 / 庆祝）
2. 如果听众只能记住一句话，应该是什么？
3. 演示结束后，你希望他们做什么？

### 受众层
4. 听众是谁？职位、背景、已知信息？
5. 他们当前对这个话题的态度？（支持 / 中立 / 怀疑）
6. 什么会让他们产生共鸣？什么会让他们反感？

### 约束层
7. 时长？页数上限？
8. 品牌/风格要求？（正式 / 活泼 / 极简 / 科技感）
9. 必须出现的元素？（logo、数据、图表、引用）
10. 必须避免的元素？

### 素材层
11. 是否有现成文档、数据、图片？
12. 是否有演讲者备注或讲稿？

## 追问策略
- 用户回答模糊时，用“能举个例子吗？”逼出具体场景
- 用户给出多个目标时，用“如果只能选一个最重要的？”做优先级排序
- 用户没有品牌约束时，提供 2-3 套风格方案让其选择

## 输出格式
```markdown
## 需求确认单
- 目标：...
- 核心一句话：...
- 受众：...
- 时长/页数：...
- 风格：...
- 必含元素：...
- 素材：...
- 风险点：...
```
