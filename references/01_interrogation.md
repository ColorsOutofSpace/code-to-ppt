# 认知访谈（拷打引擎）

## 目标
把用户模糊的“帮我做个 PPT”变成清晰、可执行的设计约束，同时开始构建叙事骨架——用金字塔原理思考：一个核心结论 → 3-5 个支撑理由 → 每页一个证据。

## 必问问题清单

### 战略层
1. 这场演示的最终目标是什么？（说服 / 汇报 / 教学 / 融资 / 庆祝）
2. 如果听众只能记住一句话，应该是什么？
3. 演示结束后，你希望他们做什么？

### 受众层
4. 听众是谁？职位、背景、已知信息？
5. 他们当前对这个话题的态度？（支持 / 中立 / 怀疑）
6. 什么会让他们产生共鸣？什么会让他们反感？

### 约束层
7. 时长？页数上限？
8. 品牌/风格要求？（正式 / 活泼 / 极简 / 科技感）
9. 必须出现的元素？（logo、数据、图表、引用）
10. 必须避免的元素？

### 素材层
11. 是否有现成文档、数据、图片？
12. 是否有演讲者备注或讲稿？

## 追问策略
- 用户回答模糊时，用“能举个例子吗？”逼出具体场景
- 用户给出多个目标时，用“如果只能选一个最重要的？”做优先级排序
- 用户没有品牌约束时，提供 2-3 套风格方案让其选择
- 用户说“介绍我们公司”这类空泛需求时，用“听众听完最想记住的一句话是什么？”逼出核心论点

## 叙事构建（访谈中同步进行）

访谈时不只是收集信息，要开始构建金字塔：
1. **核心结论**：这场演示的最终论断是什么？（一句话）
2. **顶层理由**：支撑这个结论的 3-5 个关键论点是什么？
3. **证据/故事**：每个论点需要什么数据、案例、图表来支撑？

> 不要在访谈结束后才想结构。边问边在脑中搭建金字塔，让后续大纲自然涌现。

## 输出格式
```markdown
## 需求确认单
- 目标：...
- 核心一句话：...
- 受众：...
- 时长/页数：...
- 风格：...
- 必含元素：...
- 素材：...
- 风险点：...

## 金字塔结构（访谈中同步构建）
- 核心结论：...
- 顶层理由 1：... → 支撑证据：...
- 顶层理由 2：... → 支撑证据：...
- 顶层理由 3：... → 支撑证据：...
```
