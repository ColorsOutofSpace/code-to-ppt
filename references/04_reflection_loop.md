# 反思迭代循环

## 循环规则
1. 用 `references/03_adversarial_review.md` 生成审查报告
2. 只处理 High 优先级问题
3. 修复后，重新运行完整审查
4. 重复直到审查报告中无 High 问题

## 修复优先级
1. 内容错误 / 事实错误
2. 逻辑断层
3. 受众理解障碍
4. 视觉一致性破坏
5. 信息密度失衡

## 停止条件
满足以下全部条件时停止迭代：
- 无 High 问题
- Medium 问题 ≤ 3
- 用户明确说“可以了”

## 防止过度迭代
- 单轮迭代只改审查报告列出的问题
- 不要借迭代机会重写整个 PPT
- 每轮迭代后向用户汇报改了什么
