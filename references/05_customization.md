# 用户个性化标准协商与保存

## 何时协商
在阶段 5（全量对抗审查）之前，必须先问用户：

> “默认审查维度是：内容(25)、叙事(25)、视觉(25)、密度(25)。你是否有特别关注的方面？例如：品牌一致性、数据准确性、投资人视角、演讲者友好度等。”

## 协商规则
- 如果用户说“没有”或“默认就好” → 用基线 4 维度
- 如果用户提出自定义维度 → 把它加入评分表，权重从 4 个基线维度中匀出（每个基线维度让出 5 分以内）
- 不要无限制增加维度，最多 6 个维度，总权重仍是 100
- 用户强调的维度权重 ≥ 20，其余自动下调

## Memory 管理（严格约束，覆盖所有偏好）

Memory 文件保存**所有用户确认过的偏好**，不限于审查标准，包括：
- 品牌/视觉偏好（主色、辅助色、字体、风格）
- 审查维度与权重
- 历史反馈（用户明确说过的“不要/要…”）
- 演讲者习惯（语速、是否依赖备注）

### 硬上限
- 单个 memory 文件 ≤ **5KB**（约 100 行）
- 超过上限时，必须压缩后再追加

### 压缩策略（超过上限时执行）
1. 按主题合并：把多条分散的同主题记录合并为一条长期配置
   - 多条 `## 审查偏好记录` → 合并为 `## 长期审查配置`
   - 多条 `## 视觉偏好记录` → 合并为 `## 长期视觉配置`
   - 多条 `## 用户反馈` → 合并为 `## 长期反馈摘要`
2. 每个主题只保留最近 **3 条** 原始记录的摘要，删除详细过程
3. 删除 90 天前的记录
4. 合并后的长期配置只保留：关键事实、稳定偏好（一句话），不保留每次对话的上下文

### 保存格式（精简版）
每次确认用户偏好后，在 `~/.config/code-to-ppt/memory/{user_id}.md` 追加：

```markdown
## 视觉偏好记录
- 日期：2026-06-30
- 主色：#1a365d | 辅助色：#f7fafc | 强调色：#c53030
- 字体：标题 Microsoft YaHei，正文 Segoe UI
- 用户核心要求：深色背景，科技感

## 审查偏好记录
- 日期：2026-06-30
- 维度调整：数据准确性 +25（从内容匀出）
- 用户核心要求：检查数据来源

## 用户反馈
- 日期：2026-06-30
- 反馈：不喜欢渐变背景，偏好纯色
```

如果某主题已有 3 条以上记录，先合并为：

```markdown
## 长期视觉配置
- 主色：#1a365d | 辅助色：#f7fafc | 强调色：#c53030
- 字体：标题 Microsoft YaHei，正文 Segoe UI
- 稳定偏好：深色科技感
- 最后更新：2026-06-30

## 长期审查配置
- 内容：20 | 叙事：25 | 视觉：20 | 密度：20 | 数据准确性：25
- 稳定偏好：重视数据准确性
- 最后更新：2026-06-30

## 长期反馈摘要
- 不喜欢渐变背景，偏好纯色
- 每页最多 3 个要点
- 最后更新：2026-06-30
```

## 下次加载
读取 memory 文件时，优先应用所有 `## 长期*配置`，其次读取最近一条未合并的原始记录。

开场白示例：
> “根据你的历史设置：本次视觉方案为深色科技感（#1a365d 主色），审查维度为内容(20)、叙事(25)、视觉(20)、密度(20)、**数据准确性(25)**。”
