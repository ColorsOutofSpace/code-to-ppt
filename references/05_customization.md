# 用户个性化标准协商与保存

## 何时协商
在阶段 5（全量对抗审查）之前，必须先问用户：

> “默认审查维度是：内容(25)、叙事(25)、视觉(25)、密度(25)。你是否有特别关注的方面？例如：品牌一致性、数据准确性、投资人视角、演讲者友好度等。”

## 协商规则
- 如果用户说“没有”或“默认就好” → 用基线 4 维度
- 如果用户提出自定义维度 → 把它加入评分表，权重从 4 个基线维度中匀出（每个基线维度让出 5 分以内）
- 不要无限制增加维度，最多 6 个维度，总权重仍是 100
- 用户强调的维度权重 ≥ 20，其余自动下调

## Memory 管理（严格约束）

### 硬上限
- 单个 memory 文件 ≤ **5KB**（约 100 行）
- 超过上限时，必须压缩后再追加

### 压缩策略（超过上限时执行）
1. 把多条分散的 `## 审查偏好记录` 合并为一条 `## 长期审查配置`
2. 只保留最近 **3 条** 审查偏好记录的摘要，删除详细过程
3. 删除 90 天前的记录
4. 合并后的长期配置只保留：维度、权重、用户核心要求（一句话），不保留每次对话的上下文

### 保存格式（精简版）
协商完成后，在 `~/.config/code-to-ppt/memory/{user_id}.md` 追加：

```markdown
## 审查偏好记录
- 日期：2026-06-30
- 维度调整：数据准确性 +25（从内容匀出）
- 用户核心要求：检查数据来源
```

如果已有 3 条以上记录，先合并为：

```markdown
## 长期审查配置
- 内容：20 | 叙事：25 | 视觉：20 | 密度：20 | 数据准确性：25
- 用户稳定偏好：重视数据准确性
- 最后更新：2026-06-30
```

## 下次加载
如果记忆文件存在此用户的审查偏好：
- 读取 `## 长期审查配置` 优先应用
- 如无长期配置，读取最近一条 `## 审查偏好记录`
- 开场白变为：“根据你的历史设置，本次审查维度为：内容(20)、叙事(25)、视觉(20)、密度(20)、**数据准确性(25)**”
